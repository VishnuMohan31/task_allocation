## 2. Configure the application

All settings live in `.env` at the project root. Open it and fill in your values before starting.

| Variable         | Default                  | Required | Description                                      |
|------------------|--------------------------|----------|--------------------------------------------------|
| `APP_NAME`       | `Productivity Agent API` | No       | Title shown in the `/docs` page                  |
| `LOG_LEVEL`      | `INFO`                   | No       | `DEBUG`, `INFO`, `WARNING`, or `ERROR`           |
| `STORE_PATH`     | `data/store.json`        | No       | Path to the JSON file used as the task data store |
| `OPENAI_API_KEY` | â€”                        | **Yes**  | Your OpenAI API key â€” get one at [platform.openai.com/api-keys](https://platform.openai.com/api-keys) |
| `OPENAI_MODEL`   | `gpt-4o`                 | No       | Any OpenAI chat model, e.g. `gpt-4o`, `gpt-4-turbo`, `gpt-3.5-turbo` |

Minimal `.env` to get started:
```env
OPENAI_API_KEY=sk-your-key-here
```

> The `data/` folder already exists. The store file is created automatically on the first request.

## 5. Smoke test the happy path

**Create a task**
```bash
curl -X POST http://localhost:8000/tasks \
  -H "Content-Type: application/json" \
  -d '{"title": "Write docs", "priority": 4}'
```
Expected: `201` with a task object including a UUID `id`.

**Get agent decision** (calls OpenAI under the hood)
```bash
curl http://localhost:8000/agent/decision
```
Expected: `200` with `next_task`, `productivity_score`, and `suggestion` â€” all generated by the LangChain + GPT-4o agent.

**Complete a task** (replace `<id>` with the UUID from the create response)
```bash
curl -X POST http://localhost:8000/tasks/<id>/complete
```
Expected: `200` with `completed: true` and `completed_at` set.

---

## 6. How the agent works

`GET /agent/decision` triggers a LangGraph ReAct agent backed by OpenAI. The agent:
1. Calls the `get_pending_tasks` tool to inspect your task list
2. Calls the `calculate_productivity_score` tool to compute a weighted score
3. Returns a JSON decision with the best next task, a productivity score, and an actionable suggestion

To swap the model, change `OPENAI_MODEL` in `.env` â€” no code changes needed.
